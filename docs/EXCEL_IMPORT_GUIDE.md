# Excel 自动录入数据用户指南

## 概述
Excel 自动录入数据功能支持将 Excel 文件中的数据自动导入到数据库，支持四种数据类型：
- **项目数据 (Project)** - 独立导入，无需关联项目
- **市场数据 (Market)** - 关联到指定项目
- **工程数据 (Engineering)** - 关联到指定项目
- **财务数据 (Finance)** - 关联到指定项目

## API 端点

### 1. 导入项目数据
```
POST /api/excel/import/project
Content-Type: multipart/form-data

参数:
  - file: Excel文件 (.xlsx 或 .xls)
  
响应:
{
  "status": "success",
  "message": "项目数据已成功导入"
}
```

### 2. 导入市场/工程/财务数据
```
POST /api/excel/import/{project_id}?data_type={data_type}
Content-Type: multipart/form-data

参数:
  - project_id: 项目UUID
  - data_type: "market" | "engineering" | "finance"
  - file: Excel文件 (.xlsx 或 .xls)

响应:
{
  "status": "success",
  "message": "Excel数据已成功导入【项目名称】的{data_type}板块",
  "project_id": "xxx",
  "project_name": "xxx"
}
```

## Excel 文件格式要求

### ⭐ 重要：列名必须使用中文，与前端表格保持一致

Excel 文件的列名应该使用**中文表头**，这样与前端界面的表格列名保持一致，提升用户体验。

## 功能特性

### ✅ 自动列名匹配
- Excel 列名与数据库字段名**直接对应**，无需配置映射
- 支持列的顺序任意排列
- 自动忽略多余的列

### ✅ 智能类型转换
- 自动将字符串数字转换为整数 (如 "123" → 123)
- 自动将字符串转换为小数 (如 "123.45" → Decimal(123.45))
- 自动处理空值/空字符串 (转为 NULL)
- 支持浮点数到整数的转换 (如 "123.9" → 123)

### ✅ 详细的错误报告
导入失败时，返回包含以下信息的错误：
- 行号 (Excel中的实际行号，从1开始)
- 字段名
- 错误原因和提示

示例：
```
Excel解析失败：
第2行 month：输入应为整数,但接收到的是"abc"
第3行 building_area：数值太大
```

### ✅ 权限校验
- 项目导入：需要当前用户为已登录活跃用户
- 市场/工程/财务导入：需要具有对该项目的编辑权限

### ✅ 批量导入
- 支持一次性导入多行数据 (数百行甚至数千行)
- 失败不影响成功记录的提交
- 返回成功统计数和失败统计数

### ✅ 自动关联信息
- 市场/工程/财务数据：自动关联 project_id
- 所有数据：自动关联 created_by 用户信息
- 自动记录创建时间 (created_at)

## 错误处理

### 常见错误及解决方案

| 错误 | 原因 | 解决方案 |
|---|---|---|
| 不支持的Excel格式 | 文件不是 .xlsx 或 .xls | 使用 Excel 2007+ 格式保存 |
| Excel文件过大 | 文件大小超过限制 | 分割文件为多个小文件导入 |
| 项目不存在 | project_id 错误 | 检查 project_id 是否正确 |
| 无权限编辑该项目 | 当前用户没有项目编辑权限 | 使用有权限的用户账号 |
| 输入应为整数 | 整数字段包含非数字 | 检查 year, month, floors 等字段 |
| 数值太大 | 数字超过字段允许范围 | 检查金额、比例等数值范围 |

## 最佳实践

1. **测试导入**: 先用少量数据测试，确保格式正确
2. **备份数据**: 导入前备份现有数据
3. **验证数据**: 导入后检查导入的记录是否正确
4. **分批导入**: 大批量数据建议分成多个小文件导入
5. **保持一致**: 使用统一的日期格式 (YYYY-MM-DD)
6. **填充必填项**: 确保所有必填字段都有值

## 技术实现细节

### 核心组件

1. **excel_parser.py** - ExcelParser 类
   - 解析 Excel 字节流/文件
   - 自动数据类型转换
   - 详细错误收集

2. **excel_service.py** - ExcelService 类
   - 文件校验和权限检查
   - 批量数据导入到数据库
   - 事务处理

3. **excel_router.py** - FastAPI 路由
   - HTTP 端点定义
   - 请求参数校验
   - 响应格式化

### 数据流

```
Excel 文件
    ↓
[验证] (后缀, 大小)
    ↓
[解析] ExcelParser (行→Schema校验)
    ↓
[转换] (类型转换, 数据映射)
    ↓
[入库] (生成模型, 添加关联信息, 事务提交)
    ↓
返回结果 (成功/失败信息)
```
